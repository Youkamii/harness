# resume — 이전 작업 이어받기

1. `.harness/progress.md`를 읽는다. 가장 최근 항목의 "다음 할 일"과 "주의사항"을 파악한다.
2. `.harness/features.json`이 있으면 읽고, `failing` 상태인 항목들을 확인한다.
3. `git log --oneline -15`로 최근 커밋 흐름을 확인한다.
4. 위 정보를 종합해 현재 상태를 한 단락으로 요약해 사용자에게 보여준 뒤, 다음 작업 하나를 시작한다.

## 규칙

- 이전 세션의 "완료" 주장을 그대로 믿지 않는다. 이어서 작업하기 전에 핵심 가정(빌드가 되는지, 테스트가 통과하는지)을 빠르게 재검증한다.
- 진행 파일이 없으면 사용자에게 무엇을 이어받을지 물어본다.
