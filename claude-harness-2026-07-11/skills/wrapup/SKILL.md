# wrapup — 작업 마무리

다음 세션(또는 미래의 나)이 커밋 로그와 이 보고만 읽고 이어받을 수 있게 만든다.

## 절차

1. **변경 요약**: 이번 작업에서 바뀐 파일/기능을 항목별로. 각 항목에 커밋 해시나 이슈 번호를 연결.
2. **검증 상태**: 항목마다 **확인됨**(무엇을 실행해서 확인했는지) / **미검증**(왜 못 했는지, 어떻게 검증하면 되는지)을 표시. 미검증을 숨기지 않는다.
3. **남은 일 이관**: 미완료 작업은 GitHub 이슈(가능하면) 또는 대상 저장소의 TODO 문서·커밋 메시지 같은 **영속 매체**로 남긴다. 세션 태스크(TaskCreate)는 세션이 끝나면 사라지므로 인계 수단이 아니다. 머릿속(과 세션)에만 있는 할 일은 유실된다.
4. **교훈 저장**: 이번 작업에서 얻은 재사용 가치가 있는 발견(사용자 선호, 함정, 환경 특이사항)은 메모리 디렉토리에 저장한다. 리포지토리가 이미 기록하는 것(코드 구조, 커밋 내역)은 저장하지 않는다.
5. **다음 시작점**: 다음 세션이 그대로 복사해 시작할 수 있는 한 문장을 제안한다. (예: "PR #12 리뷰 반영부터 이어서")

## 규칙

- 커밋 안 된 변경이 남아 있으면 커밋할지 스스로 판단해 처리하고 결과를 보고한다 (의미 단위가 아니면 WIP 커밋 대신 상태를 명시).
- 테스트가 깨진 채로 마무리하면 보고 맨 위에 그 사실을 적는다.
