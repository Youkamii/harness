# checkpoint — 진행 상황 저장

다음 세션(또는 컨텍스트 리셋 후의 나)이 이 파일들만 읽고 작업을 이어받을 수 있어야 한다.

1. `.harness/` 디렉토리가 없으면 만든다.
2. `.harness/progress.md`에 새 항목을 추가한다 (기존 내용 위에 덧붙이지 말고 아래에 누적):
   - 날짜/시각
   - **완료한 것** — 실제 실행으로 검증된 것만. 미검증이면 "미검증"이라고 명시
   - **다음 할 일** — 구체적으로, 파일 경로와 함께
   - **주의사항** — 막힌 것, 시도했다 실패한 접근, 함정
3. `.harness/features.json`이 있으면 상태를 갱신한다. 검증된 것만 `"passing"`으로 바꾼다.
4. git 저장소라면 서술적인 커밋 메시지로 커밋을 제안한다.

## 규칙

- 검증하지 않은 것을 완료로 적지 않는다.
- "거의 다 됨" 같은 모호한 표현 대신 남은 작업을 항목으로 적는다.
