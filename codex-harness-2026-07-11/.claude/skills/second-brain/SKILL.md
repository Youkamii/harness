# second-brain — 옵시디언 볼트 장기 기억

볼트 경로: `~/Documents/Lia` (환경변수 `BRAIN_VAULT`로 재정의 가능)
기억 폴더: `<볼트>/AI/메모리/` — 인덱스는 `메모리.md` (허브노트 패턴)

## 기록 (쓰기)

언제: ① 사용자가 "기억해둬"/"위키에 정리해둬"라고 하면 즉시. ② 세션이 마무리될 때 중요한 새 정보(선호, 결정, 진행 중인 일, 환경 변화)가 있으면 자동 — 단, 무엇을 기록했는지 반드시 사용자에게 보고한다.

방법:
1. `메모리.md`를 먼저 읽는다 (현재 기억 상태 파악).
2. 짧은 사실은 `메모리.md`의 해당 절(프로필/환경/진행 중인 일/기록)에 직접 추가·갱신. 긴 주제(프로젝트 노트, 정리 요청받은 내용)는 `AI/메모리/` 안에 별도 노트로 만들고 `메모리.md`에서 `[[노트명]]`으로 링크.
3. **모순 처리**: 기존 기억과 충돌하면 옛 내용을 지우지 말고 새 사실로 갱신하되 `(갱신: 날짜, 이전: ~였음)` 형태로 이력을 남긴다.
4. 날짜를 항상 붙인다. "진행 중인 일"은 끝나면 완료 표시 후 기록 절로 내린다.

## 회상 (읽기)

1. `메모리.md`는 세션 시작 시 훅(brain-recall)이 자동 주입한다 — 다시 읽을 필요 없음.
2. 작업이 사용자의 과거 기록과 관련될 때(프로젝트, 사람, 학습 주제 등): 볼트 전체에서 키워드 검색(Grep)해 관련 노트를 찾아 읽는다. 허브노트(폴더명과 같은 이름의 .md)부터 읽으면 빠르다.
3. 찾은 내용을 답에 활용할 때는 어느 노트에서 왔는지 밝힌다.

## 안전 규칙 (중요)

- **쓰기는 `AI/메모리/` 폴더 안에서만.** 볼트의 다른 폴더는 읽기 전용 — 사용자가 명시적으로 "이 노트 고쳐줘"라고 한 경우에만 예외.
- 볼트 내용을 외부 서비스로 전송하지 않는다 (개인 기록임).
- 민감 정보(비밀번호, 금전 세부, 건강)는 사용자가 명시적으로 요청할 때만 기록한다.
- 기록 후에는 항상 "○○를 메모리에 기록했다"고 한 줄 보고한다 — 몰래 쌓지 않는다.
