## Synthesis

The highest-confidence design is not “more agents.” It is a small authoritative controller plus narrowly privileged agents, durable replayable state, and machine-verifiable exit conditions. Popularity in this strict 30-day niche is still thin; therefore the implementation favors convergent patterns and direct adversarial tests over star count alone.

The resulting harness deliberately combines:

- Codex-native plugin and reusable skills for the user-facing workflow;
- a TypeScript controller as the only Git, GitHub, state, retry, and completion authority;
- GitHub issues as requested work records, not as an untrusted command channel;
- one worktree and owned path set per feature;
- exact Codex thread persistence rather than “last session” guessing;
- machine checks before commit and again after integration;
- two independent reviewers with different attack objectives;
- a final gate bound to the current integrated content and plan hashes;
- explicit non-goals so autonomous permission cannot silently broaden the task.

Not adopted: tmux-only orchestration, cross-model dependencies, mandatory databases/vector stores, dashboards, deployments, force pushes, automatic remote merges, recursive leaf-agent spawning, or arbitrary shell hooks. These either add an unrequested dependency, weaken portability, or expand authority beyond the requested coding outcome.
